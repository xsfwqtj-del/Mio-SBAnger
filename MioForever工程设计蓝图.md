### 〇.2 模型分工

```
pro 模型 (主推理)                   flash 模型 (辅助 + 后台)
─────────────────────              ─────────────────────────
[4] LLM 推理 (核心)               [3] 烬织检索精选
                                   [5b] 反思轮次 (flash · 可选)
                                   [6c] V3 一致性 · V4 结果回验 (可选)
                                   [后台·快] session-to-memory 提取

```

### 〇.3 三个回环

| # | 类型 | 路径 | 触发条件 |
|---|------|------|---------|
| **工具回环** | 正常循环 | [6c] → [4] | 工具执行完毕后继续推理，可能再次调工具 |
| **拦截回环** | 异常纠正 | [6a] → [4] | V1参数/V2语义/行为策略门禁拦截 |
| **判定回环** | 异常纠正 | [5] → [4] | POST_THINK BLOCK，安全/格式/合规不通过 |

三个回环之外是线性流水线 [1]→[2]→[3]→[4]→[5]→[7]→[8]→[后台]。混合架构: 主干 Plan-and-Execute + 异常 ReAct 纠正。

## 〇.B 配置即马具 — 从文件到行为的端到端链路

> 配置之于 Agent，如同马具之于马——一套让行为可控、可预期的约束系统。方向是用户拉的，马具决定了 Mio 能做什么、不能做什么、怎么跑。

### 〇.B.1 核心理念：每一份配置 = 文本引导 + 验证闭环

配置和代码不同。配置**大部分时候就是一段文本**，在 [2] BUILD 注入 system prompt，让 LLM 读到并照着做。

但光告诉 LLM 还不够——你还得检查它是否真的照做了。所以每一份配置天然是一对：

```
配置项 = 文本注入 ([2] BUILD)  +  可选验证规则 (Hook 点)
         ────────────────           ────────────────
         告诉 LLM 怎么做             检查 LLM 做了没
```

**例子：思维链。**

用户写了一段思考步骤："一检设定、二思指令、三辨视角、四遵写规"。

```
思维链 Section
    │
    ├──→ [2] BUILD → 注入 system prompt
    │      "请按以下步骤思考：一检设定 / 二思指令 / 三辨视角 / 四遵写规"
    │      LLM 读到这段文本，照着做。
    │
    └──→ [4] 流式输出时 → Hook 验证
           扫描 think 内容:
             ✗ 不含"检设定" → 打回 LLM: "你忘了第一步：检设定"
             ✗ 不含"思指令" → 打回 LLM
             ✓ 全有 → 放行
           这是同一份配置的"另一半"——不信任 LLM 靠自觉，用验证兜底。
```

**所有配置项都遵循这个配对模型：**

| 配置项 | 文本注入 ([2] BUILD) | 验证规则 (Hook 点) |
|--------|---------------------|-------------------|
| 思维链 | "一检设定、二思指令…" | [4] 流式检查: think 内是否含关键步骤 |
| 交互风格 | "回复简洁、短句、动词先行" | [5] POST_THINK: 回复长度/句式检查 |
| 行为策略 | "不要删除文件" | [6a] PRE_TOOL: 拦截 rm -rf |
| 校验回路 | "回答前自查安全性" | [6c] POST_TOOL: V3一致性检查 |
| 约束松弛度 | (无——不注入 prompt) | [6a]: strict=拦截/flexible=提醒/creative=放行 |

### 〇.B.3 穿搭

用户按配置项粒度自由拼配——思维链用配置A的，交互风格保留自己的——Pack Loader 拼出"当前激活配置"。穿搭保存时生成新配置条目，源配置不受影响。

### 〇.B.4 优先级

```
用户本地 (最高)  >  导入 .mio-pack  >  出厂 (fallback)
```

任何配置项如果用户从未设置，退回出厂默认。确保 Mio 永远不会"裸奔"。

### 〇.B.5 热切换

切换配置 → 当前轮不受影响 → ON_STOP 后 → 下一轮 BUILD Pack Loader 重新拆解+分发。

### 〇.B.6 8 Section 的定位

8 Section（用户画像/交互风格/思维方式/行为策略/校验回路/推理模板/约束松弛度/反思提示词）是当时的设计尝试。它有局限性——预先切死了 8 个维度。但它提供了一套**启动种子**：用户能从这 8 个方向开始配置，也可以自定义新的配置项。Section 不是不可变的框架，是可扩展的起点。

### 〇.B.7 关键决策

| 决策 | 说明 |
|------|------|
| 配置=文本+验证 | 每一项配置天然是一对：系统提示词注入 + 可选 Hook 验证规则 |
| 文本是第一位的 | 大多数配置就是一段文本，[2] BUILD 注入。不需要复杂的"分发机制" |
| 验证是兜底的 | 不信任 LLM 靠自觉。关键行为用 Hook 点验证 |
| Section 可扩展 | 8 Section 是启动种子，用户可新增配置项 |
| 约束松弛度不注入 prompt | 纯代码级参数，LLM 不需要感知 |
| 校验回路属 Mio 行为引擎 | 烬织只做记忆的归纳整理 |
| 出厂配置是 fallback | 任何配置项未设置时退回出厂默认 |
| 本地配置不打包 | 用户的私有配置不分享、不版本化 |

### 4.4 记忆网（MAGMA 四图，论文 v2）

**施工状态**：📐 设计立定，待施工
**依据**：MAGMA 论文 v2（arXiv:2601.03236, ACL 2026 Main，2026/04/07 接收）
**取代**：ADR-0002 实体分类锚点图（#652 方向偏差：实体标签 ≠ 事实锚点）

> 记忆网是**一张图**，四种边。不是四个独立的图——同一个事件节点可以同时有语义边、时间边、因果边、实体边。

**四边定义**：

| 边类型 | 什么意思 | 怎么建 | 谁建 |
|--------|---------|--------|------|
| 时间边 | "A 发生在 B 之前" | 新碎片追加到时间线末尾，不可变 | 快通道，全自动 |
| 语义边 | "A 和 B 内容相似" | 向量 cos > 阈值 → 连边；同会话邻近窗口 → 连边 | 快通道，全自动 |
| 因果边 | "A 导致了 B" / "A 是 B 的前提" / "A 回答了 B" | 慢通道后台 worker：取新事件 2 跳邻域 → flash LLM 推断 → 写入 | 慢通道，flash LLM |
| 实体边 | "A 和 B 说的是同一个人/物" | 快通道标实体名；慢通道 LLM 合并同义实体（"React" = "reactjs"） | 快通道标 + 慢通道合 |

**构建流水线**：

```
每轮对话结束
    │
    ├──→ 快通道（同步，毫秒级，不调 LLM）
    │     ① session-to-memory 提取碎片 → 事件节点
    │     ② 追加时间边（连到上一事件）
    │     ③ 计算向量 → 语义边（相似度 > 阈值）
    │     ④ 提取实体名（标注在碎片上）
    │     ⑤ 事件 ID 入队列 → 触发慢通道
    │
    └──→ 慢通道（后台 worker，调 flash LLM）
          loop:
            从队列取事件 ID
            取 2 跳邻域（时间边 + 语义边自然跨会话）
            flash LLM 看邻域上下文，推断：
              → 因果边（LEADS_TO / BECAUSE_OF / ENABLES / PREVENTS / RESPONSE_TO）
              → 实体边（同义实体合并 + 跨会话同一实体链接）
            写入图
            4.5 反馈闭环后续验证（被引用强化 / 被否认归零）
```

**时间边为什么能跨会话**：时间 backbone 是一条连续不可变链，不受会话边界影响。慢通道 worker 取 2 跳邻域时，顺着时间边自然跨过会话边界，LLM 看到的是跨会话的上下文。

**检索时怎么用**（对接 4.2）：

```
用户问 →
  ① 判断意图（问"为什么"？问"什么时候"？问"谁"？）
  ② 意图 → 四边权重（因果问题加重因果边、时间问题加重时间边…）
  ③ 锚点识别（向量 + 关键词 + 时间窗 三信号融合）
  ④ 沿加权边扩散（Beam Search，算结构对齐 + 语义相关双分）
  ⑤ 拓扑排序 + 溯源标注 → 拼入 prompt
```

- 导入不覆盖，多配置在配置池中平等共存
- 挂载 = 选择一份配置激活（默认整份，可穿搭）
- 穿搭 = 按 8 Section 粒度从不同配置中取片段自由组合
- 穿搭结果可保存为新配置 → 导出 → 分享（中阶用户的核心创作路径）
