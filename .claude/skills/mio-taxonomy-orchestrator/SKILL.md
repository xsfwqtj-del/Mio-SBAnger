# Mio Taxonomy Orchestrator — 配置统筹

> 分类学技能 1。Agent 行为配置的总入口——检查覆盖、诊断盲区、路由修复。

## 执行流程

### 步骤 1: 读取用户配置

读取以下文件（如果存在）:
- 全局 CLAUDE.md: `C:\Users\xsfwq\.claude\CLAUDE.md`
- 当前项目 CLAUDE.md: `.\CLAUDE.md`
- 全局记忆索引: `D:\claude-memory\MEMORY.md`

### 步骤 2: 运行 33 题自检

对照 `data/questionnaire.json`，逐维检查用户当前配置覆盖了哪些。

输出格式:
```
## 配置覆盖矩阵

### 通用维度
  行为策略:      [████████░░] 80% — 有安全红线，缺少灰色地带规则
  校验回路:      [███░░░░░░░] 30% — 有 Hook，缺 V2/V3/V4 + 回环终止
  推理模板:      [██████████] 100% — 覆盖完整
  工具可用性:    [██░░░░░░░░] 20% — 有 Hook，缺工具清单+纪律+MCP配置
  技能触发规则:  [█░░░░░░░░░] 10% — 有技能但触发条件未定义

  反思提示词:    [██████░░░░] 60% — 有"回答前检查"，可系统化为问题集
```

### 步骤 3: 可选 — 调 session-to-memory

如果用户说"从最近的错误中学习"或"检查最近的教训"，调 session-to-memory 技能提取最近的"错误/偏好/模式"类记忆，按 `data/memory-to-dimension.json` 映射到 9 维度，标注哪些教训尚未固化为配置规则。

### 步骤 4: 输出建议 + 路由

```
## 优先修复
1. [P0] 工具可用性 — 没有工具纪律 → 建议调 writer 补充
2. [P0] 校验回路 — 缺少回环终止条件 → 建议调 tools-config 加 Hook
3. [P1] 用户画像 — 未显式声明 → 建议调 writer 生成画像模板

## 已覆盖
- 思维方式: 完整，保持即可
- 交互风格: 完整，可考虑加反例提高遵从率
- ...

## 路由建议
- 要补充配置文本 → 说"帮我写 {维度名}"
- 要配工具/Hook → 说"帮我配 {工具类型}"
- 要查错误 → 说"查看错误对照表"
- 要对工作流 → 说"查看 {场景} 工作流"
```

## 关键原则

- **不替用户改配置**：只给建议，让用户确认。不直接编辑 CLAUDE.md
- **优先安全盲区**：行为策略和校验回路的盲区 > 风格盲区
- **少即是多**：如果用户已有 7/9 覆盖，补充盲区即可，不画蛇添足
- **可追溯**：每条建议标注对应的分类学维度 + 问卷题号

## 迭代闭环

每次用户纠正 Agent 的错误 → 是一次迭代机会：

```
Agent 犯错 → 用户纠正
  → 问: "要不要把这次的教训写进配置？"
  → 用户说"要" → 调 writer → 更新 CLAUDE.md
  → 如果是可规则化的错误 → 调 tools-config → 加 Hook
  → 如果是新类型的错误 → 建议纳入 error-diagnosis.md
```

不需要复杂的自动化——每次纠正就是一次迭代。

## 边界

**负责**:
- 读取用户 CLAUDE.md + 运行 33 题自检
- 输出覆盖矩阵（哪个维度有/缺/弱）
- 路由到正确的子技能或文档
- 建议迭代（从最近的错误中学习）

**不负责**:
- 不直接写配置文本 → 交给 writer
- 不配置 MCP/Hook → 交给 tools-config
- 不搜索新工具 → 交给 discovery
- 不直接编辑用户的 CLAUDE.md

**交接**:
- 缺内容维度 → "建议调 writer: 帮我写 {维度名}"
- 缺工具配置 → "建议调 tools-config: 帮我配 {工具类型}"
- 要搜工具 → "建议调 discovery: 帮我搜 {领域} 的工具"
- 症状诊断 → "建议先查错误对照表: error-diagnosis.md"
