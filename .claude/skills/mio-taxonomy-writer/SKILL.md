# Mio Taxonomy Writer — 写 CLAUDE.md

> 分类学技能 2。按维度生成可直接粘贴进 CLAUDE.md 的配置文本。

## 触发条件

- 用户说"帮我写 XXX 的配置"（XXX = 9 维度之一）
- 统筹技能路由过来，指定了目标维度
- 用户说"优化 CLAUDE.md"但没指定维度 → 先看用户当前配置，再建议优先优化的维度

## 执行流程

### 1. 确定目标维度

如果用户没指定维度，先读用户的 CLAUDE.md（全局 + 项目），给出快速诊断：
"你的配置覆盖了这些维度：A/B/C。缺少：D/E/F。建议优先补充 D，因为..."

### 2. 读参考数据

按目标维度，读对应参考：
- **逐维指南**: `references/per-dimension-guide.md`
- **编写方法论**: `D:\Mio-SBAnger\docs\taxonomy\methodology\claude-md-writing-guide.md`
- **指令有效性**: `D:\Mio-SBAnger\docs\taxonomy\methodology\instruction-effectiveness.md`

### 3. 生成配置文本

按标准模板输出：

```markdown
## 维度: {维度名}

### 推荐配置
{填好的配置文本，可直接粘贴}

### 为什么这样写
- 每一条为什么有效（不是形容词，是行为指令）
- 和常见错误写法的对比

### 验证方式
- 怎么检查这条配置生效了？
- {规则匹配 / LLM 判定 / 行为拦截 / 无}

### 和其他维度的关系
- 这条配置和哪些已有规则可能冲突？
- 优先级建议: [覆盖] / [补充] / [建议]
```

### 4. 确认

输出后问用户：是否满意？要不要调？确认后提醒用户粘贴到对应位置（全局/项目 CLAUDE.md）。

## 关键原则

- **每条指令必须是行为式**：不说"严谨"，说"先审查再动手"
- **每条禁令配替代方案**："不要 X"后面跟"应该 Y"
- **≤5 条规则**：一个维度超过 5 条 → 建议拆分或降优先级
- **带验证建议**：告诉用户怎么确认这条规则真的生效了
- **用反例**：每条指令最好配一个"不要这样"的例子

## 注意事项

- 不替用户决定配置内容——只给推荐，让用户选
- 如果用户的现有配置已经很好，不要画蛇添足
- 不同模型对同一配置的遵从率不同——如果是关键安全规则，建议同时加 Hook 拦截

## 边界

**负责**:
- 按 9 维度生成 CLAUDE.md 配置文本
- 提供每个维度的有效写法 vs 无效写法对照
- 给出验证方式建议
- 标注优先级标签和与其他维度的关系

**不负责**:
- 不配置 MCP/Hook → 交给 tools-config
- 不搜索新工具/技能 → 交给 discovery
- 不直接编辑用户的 CLAUDE.md（只输出文本让用户粘贴）
- 不做通用写作（文章/文档）→ 那是写作技能的事
- 不替用户做配置决策 → 只给推荐

**交接**:
- 配置涉及工具纪律 → "建议调 tools-config: 帮我配 {工具类型} 的权限"
- 配置涉及未知工具 → "建议调 discovery: 帮我搜 {领域} 的工具"
- 不确定用户偏好 → 问用户，不猜
