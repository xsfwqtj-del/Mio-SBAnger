# 逐维度写法指南

> 每个维度的: 核心指令模板 + 有效 vs 无效 + 样例

## 行为策略

**模板**:
```
安全红线:
  - 绝不: {具体操作} → {替代方案}
操作纪律:
  - {行为要求} → 违反: {BLOCK/CONFIRM/FLAG}
```

**有效**: 每条禁令有替代方案 | **无效**: 只有"注意安全"

## 校验回路

**模板**:
```
PRE_TOOL: V1 {参数检查} + V2 {语义合理性}
回环终止: 工具 max={N}，拦截 max={N}
```

**有效**: 有具体规则和终止条件 | **无效**: 只有"检查一下"

## 推理模板

**模板**:
```
{任务类型}: {顺序结构}
  方案类: 结论 → 理由 → 风险
  施工类: 步骤概览 → 逐步: 做什么·产出·验证
```

**有效**: 按任务类型分模板 | **无效**: "结构清晰"

## 工具可用性

**模板**:
```
{工具名}:
  适用: {场景}
  禁用: {场景}
  纪律: {限制}
  权限: allow / deny / ask
```

**有效**: 每个工具有适用/禁用 + 纪律 | **无效**: 只列工具名

## 技能触发规则

**模板**:
```
{技能名}:
  触发: {关键词/语义/手动}
  不触发: {排除场景}
  优先级: {高/中/低}
  恢复: {立即/会话级}
```

## 用户画像

**模板**:
```
角色: {你的角色}
技术栈: {偏好}
当前项目: {项目 + 阶段}
语言: {中文/English}
```

**有效**: 只有可操作信息 | **无效**: 简历式无关信息

## 交互风格

**模板**:
```
语气: {直接/委婉/正式/随意}
长度: {先结论后展开/先过程后结论}
不: {不要的列表}
```

**有效**: 每条件有反例 | **无效**: "简洁专业"

## 思维方式

**模板**:
```
原则: {判断标准} → {当满足时做什么}
启发式: {具体规则}
不确定时: {默认行为}
```

**有效**: 有具体行为和反例 | **无效**: "理性严谨"

## 反思提示词

**模板**:
```
回答前检查:
  1. {具体可回答的问题}
  2. ...
```

**有效**: 每条可回答是/否 + 答案=否时有纠错出口 | **无效**: "检查质量"
