> 版本：v1.0
> 日期：2026-06-11
> 状态：设计草案

---

## 一、项目概述

### 1.1 核心命题

当前 LLM 助手的根本问题不是"模型不够强"，而是**用户没说清楚，模型听错了意思**。

意图转译网络（ITN）只做一件事：**把用户的自然语言输入，通过集体智慧转译成 LLM 最能稳定执行的结构化指令**。

它不是配置系统，不是记忆系统，而是一个**分布式意图匹配网络**——每个用户的输入被抽象为"场景签名"，在云端匹配历史上相同或相似场景下最成功的转译方式，返回最优解。

### 1.2 与现有方案的本质区别

| 方案 | 核心逻辑 | 问题 |
|------|---------|------|
| 提示词工程工具 | 优化单条 prompt 的表达 | 假设用户已经知道怎么写 |
| 角色卡/配置系统 | 静态注入预设行为 | 一刀切，不随场景变化 |
| 个人记忆系统 | 记录用户偏好 | 只解决"你是谁"，不解决"这句话什么意思" |
| **意图转译网络** | **场景匹配 + 集体智慧** | **解决"这句话在这个场景下最该怎么说"** |

### 1.3 价值假设

> 当用户说"写个爬虫"时，真正想说的东西在 90% 的场景下已经被其他人在相似场景下说过了。我们不需要让用户学会"怎么说清楚"，只需要帮他们找到"最像自己此刻的那个成功先例"。

## 二、核心概念

### 2.1 场景签名（Scene Signature）

场景签名是用户输入的**匿名化、结构化指纹**。它不是用户说了什么，而是**用户在什么情境下问了什么类型的问题**。

```yaml
scene_signature:
  # 核心维度（必须）
  intent: "code_generation"           # 意图分类
  domain: "web_scraping"              # 领域标签
  complexity: "medium"                # 复杂度

  # 上下文维度
  temporal_context: "present"         # 时间上下文
  causal_chain:                       # 前因链（最近3轮意图）
    - "information_retrieval"
    - "code_review"
    - "code_generation"
  tone: "neutral"                     # 语气

  # 环境维度
  model_family: "claude-3.5"          # 目标模型
  language: "zh"                      # 用户语言

  # 派生标识
  signature_hash: "a3f7e2d9..."       # 六维度哈希（用于精确匹配）
```

**关键原则**：签名不上传原始输入，只上传结构化特征。即使云端数据库泄露，也无法反推用户说了什么。

### 2.2 转译结果（Translation Result）

转译结果是发给 LLM 的**结构化指令**，不是自然语言扩写。

```yaml
translation_result:
  result_id: "uuid"
  structured_prompt:
    task_type: "code_generation"
    requirements:
      - "使用 Python 语言"
      - "包含异常处理"
      - "包含反爬机制"
      - "输出完整可运行脚本"
    constraints:
      - "不使用第三方库（除非标准库无法解决）"
      - "添加详细注释"
    context:
      user_expertise: "intermediate"
      previous_similar_tasks: ["task_0042"]
    output_format: "full_script_with_comments"

  # 元数据
  source: "community"          # community | personal | fallback
  author_id: "anon_hash"       # 匿名作者标识
  created_at: "2026-06-01"

  # 统计
  usage_count: 152
  avg_feedback: 1.8            # 平均反馈分（-2 到 +2）
  confidence: 0.87             # 系统置信度
```

### 2.3 匹配度（Match Score）

匹配度衡量"当前场景"和"历史场景"的相似性，不是字符串相似，而是**语义维度相似**。

```
match_score = Σ(dimension_weight × dimension_match)

维度权重：
  intent:      0.40   # 意图必须一致，权重最高
  domain:      0.30   # 领域必须一致
  complexity:  0.10   # 复杂度影响输出规模
  temporal:    0.10   # 时间上下文影响措辞
  causal_chain: 0.07  # 前因链影响连续性
  tone:        0.03   # 语气微调输出风格
```

### 3.2 数据流说明

**阶段一：本地提取（同步，<100ms）**
1. 用户输入进入本地提取器
2. 提取六维度特征，生成场景签名
3. 签名哈希化，不上传任何原始文本

**阶段二：匹配查询（异步，<500ms）**
1. 本地先查个人缓存（命中则跳过云端）
2. 未命中则上传签名到云端匹配引擎
3. 云端返回 top-3 转译结果

**阶段三：本地决策（同步，<50ms）**
1. 如果有多个候选，展示给用户选择（或按好评度自动选）
2. 格式化器把结构化指令转译为当前 LLM 的最优 prompt 格式

**阶段四：LLM 调用**
1. 格式化后的 prompt 发给 LLM
2. 流式输出给用户

**阶段五：反馈收集（异步，不阻塞）**
1. 追踪用户行为（编辑/重试/直接执行）
2. 匿名化后上报反馈，更新云端统计

## 四、本地层详细设计

### 4.1 输入捕获层

**职责**：捕获用户输入，预处理文本。

```python
class InputCapture:
    def capture(self, raw_input: str, context: DialogContext) -> UserInput:
        """
        - 截断超长输入（>4000 字符先摘要）
        - 编码标准化（统一 UTF-8）
        - 注入对话上下文（最近 3 轮）
        """
        return UserInput(
            text=raw_input,
            dialog_history=context.last_n_turns(3),
            timestamp=now(),
            platform=context.platform
        )
```

### 4.2 场景签名生成器

**核心约束**：不用 LLM，全部本地规则/轻量模型完成。

#### 4.2.1 意图分类器（Intent Classifier）

**实现**：本地运行的轻量 BERT 模型（<30MB）。

**意图清单**（可扩展）：

```yaml
intents:
  - code_generation        # 生成代码
  - code_review            # 审查代码
  - code_debug             # 调试代码
  - explanation            # 解释概念
  - comparison             # 对比分析
  - information_retrieval  # 信息检索
  - writing                # 写作辅助
  - translation            # 翻译
  - brainstorming          # 头脑风暴
  - task_planning          # 任务规划
  - casual_chat            # 闲聊
```

**置信度阈值**：
- `>0.85`：确定分类，直接采用
- `0.6-0.85`：模糊，标记为 `ambiguous`，附 top-2 候选
- `<0.6`：无法分类，标记为 `unknown`，不走云端匹配

#### 4.2.5 前因链生成器

从本地对话状态机读取最近 3 轮的意图：

```python
class DialogStateMachine:
    def get_causal_chain(self, n=3) -> List[str]:
        """
        返回最近 n 轮对话的意图序列。

        return [turn.intent for turn in self.history if turn.intent][-n:]
```

### 4.3 签名哈希生成

```python
class SignatureHasher:
    def hash(self, signature: SceneSignature) -> str:
        """
        对核心维度做确定性哈希，用于精确匹配。
        """
        core = f"{signature.intent}:{signature.domain}:{signature.complexity}:{signature.temporal_context}"
        return sha256(core.encode()).hexdigest()[:16]
```

### 5.2 匹配引擎

#### 5.2.1 分层匹配策略

```python
class MatchEngine:
    def match(self, signature: SceneSignature) -> List[TranslationResult]:
        # Level 1: 精确匹配（六维度完全一致）
        exact = self.exact_match(signature.signature_hash)
        if exact and exact.avg_feedback > 0.5:
            return exact, "exact"

        # Level 2: 核心维度匹配（intent + domain 必须一致，其他 ≥3/4 一致）
        fuzzy = self.fuzzy_match(signature, threshold=0.80)
        if fuzzy:
            return fuzzy[:3], "fuzzy"

        # Level 3: 粗粒度匹配（只匹配 intent + domain）
        coarse = self.coarse_match(signature.intent, signature.domain)
        if coarse:
            return coarse[:3], "coarse"

        # Fallback: 返回该 intent + domain 的全局默认模板
        return self.fallback(signature.intent, signature.domain), "fallback"
```

#### 5.2.2 向量索引（用于模糊匹配）

对非核心维度（complexity, temporal, causal_chain, tone）做向量化，用近似最近邻（ANN）搜索：

```python
# 每个签名生成一个 64 维向量
# intent 和 domain 用 one-hot，其他维度用 embedding
vector = concat([
    embed_intent(signature.intent),      # 16 dim
    embed_domain(signature.domain),      # 16 dim
    embed_complexity(signature.complexity),  # 8 dim
    embed_temporal(signature.temporal_context),  # 8 dim
    embed_causal(signature.causal_chain),  # 8 dim
    embed_tone(signature.tone)            # 8 dim
])

# 用 HNSW 索引（Milvus/Pinecone/自研）
neighbors = vector_index.search(vector, k=100)
# 再按 match_score 公式重排序
```

### 5.3 结果排序器

排序不是简单按好评度，而是综合指标：

```python
def score_result(result: TranslationResult, match_level: str) -> float:
    """
    综合评分 = 匹配度 × 好评度 × 新鲜度 × 置信度
    """
    match_weight = {"exact": 1.0, "fuzzy": 0.9, "coarse": 0.7, "fallback": 0.5}[match_level]

    # 好评度：-2 到 +2 映射到 0 到 1
    feedback_score = (result.avg_feedback + 2) / 4

    # 新鲜度：最近 30 天有使用记录的加分
    freshness = 1.0 if result.last_used > now() - 30_days else 0.7

    # 置信度：使用次数越多越可信
    confidence = min(result.usage_count / 100, 1.0)

    return match_weight * feedback_score * freshness * confidence
```

## 六、集体转译数据库（Schema）

### 6.1 核心表

```sql
-- 场景签名表
CREATE TABLE scene_signatures (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    signature_hash  VARCHAR(16) NOT NULL,
    intent          VARCHAR(32) NOT NULL,
    domain          VARCHAR(32) NOT NULL,
    complexity      VARCHAR(16) NOT NULL,
    temporal_context VARCHAR(16) NOT NULL,
    causal_chain    JSON,        -- ["intent1", "intent2", "intent3"]
    tone            VARCHAR(16) NOT NULL,
    model_family    VARCHAR(32) NOT NULL,
    language        VARCHAR(8)  NOT NULL,
    created_at      TIMESTAMP DEFAULT NOW(),

    INDEX idx_hash (signature_hash),
    INDEX idx_intent_domain (intent, domain),
    INDEX idx_model (model_family)
);

-- 转译结果表
CREATE TABLE translation_results (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    signature_id    BIGINT NOT NULL REFERENCES scene_signatures(id),
    result_uuid     VARCHAR(36) UNIQUE NOT NULL,

    -- 结构化 prompt（JSON）
    structured_prompt JSON NOT NULL,

    -- 元数据
    source          VARCHAR(16) NOT NULL,  -- community | personal | system
    author_hash     VARCHAR(32),           -- 匿名作者标识
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW(),

    -- 统计
    usage_count     INT DEFAULT 0,
    total_feedback  FLOAT DEFAULT 0,       -- 反馈总分累计
    feedback_count  INT DEFAULT 0,
    avg_feedback    FLOAT GENERATED ALWAYS AS (
        CASE WHEN feedback_count > 0 THEN total_feedback / feedback_count ELSE 0 END
    ) STORED,

    -- 质量指标
    confidence      FLOAT DEFAULT 0.5,
    status          VARCHAR(16) DEFAULT 'active',  -- active | trial | deprecated

    INDEX idx_signature (signature_id),
    INDEX idx_feedback (avg_feedback DESC),
    INDEX idx_usage (usage_count DESC)
);

-- 反馈记录表
CREATE TABLE feedback_records (
    id              BIGINT PRIMARY KEY AUTO_INCREMENT,
    result_id       BIGINT NOT NULL REFERENCES translation_results(id),

    -- 匿名化用户标识（用于计算用户可信度）
    user_hash       VARCHAR(32) NOT NULL,

    -- 反馈值（-2 到 +2）
    feedback_value  TINYINT NOT NULL,

    -- 反馈来源
    source_type     VARCHAR(16) NOT NULL,  -- implicit | explicit

    -- 上下文（可选）
    match_level     VARCHAR(16),           -- 当时匹配到的层级
    model_family    VARCHAR(32),           -- 当时使用的模型

    created_at      TIMESTAMP DEFAULT NOW(),

    INDEX idx_result (result_id),
    INDEX idx_user (user_hash)
);

-- 用户可信度表（用于反馈加权）
CREATE TABLE user_reputation (
    user_hash       VARCHAR(32) PRIMARY KEY,
    total_feedback  INT DEFAULT 0,
    positive_ratio  FLOAT DEFAULT 0.5,     -- 正反馈占比
    reputation_score FLOAT DEFAULT 1.0,    -- 可信度权重
    last_active     TIMESTAMP DEFAULT NOW()
);
```

### 6.2 数据规模估算

| 指标 | 估算 | 说明 |
|------|------|------|
| 单用户日均签名 | 20-50 条 | 每次输入一条 |
| 10 万用户日签名 | 200-500 万条 | 去重后约 10-20% 唯一 |
| 唯一签名库存 | 1000 万条（运行 1 年后） | 大量重复签名合并 |
| 单条转译结果 | ~2KB（JSON） | 结构化 prompt |
| 总存储（1 年）| ~100GB | 含索引和反馈记录 |

## 七、反馈与进化机制

### 7.1 隐式反馈收集

用户行为自动映射为反馈值：

| 用户行为 | 反馈值 | 说明 |
|---------|--------|------|
| 直接执行，无任何编辑 | +2 | 强正反馈 |
| 小幅编辑后执行（<20% 改动） | +1 | 弱正反馈 |
| 大幅编辑后执行（>50% 改动） | -1 | 转译方向对但细节不对 |
| 切换为另一个候选结果 | -1 | 当前结果不如另一个 |
| 删除重来，重新输入 | -2 | 强负反馈 |
| 同一签名下连续使用某结果 | +0.5/次 | 习惯性偏好（上限 +3） |

**上报时机**：会话结束后批量上报，不实时逐条发送。

### 7.2 显式反馈（可选）

在每轮输出后，提供一个极简评分入口：

```
[转译质量] 👍 正好 | 👎 不对 | ✏️ 我改了一下
```

点击即上报，无额外弹窗。

### 7.3 反馈加权

```python
def weighted_feedback(raw_feedback: float, user_hash: str) -> float:
    user = get_user_reputation(user_hash)
    weight = user.reputation_score if user else 0.5  # 新用户权重 0.5

    # 异常检测：如果某用户总是给极端值，降权
    if user and user.positive_ratio > 0.95:
        weight *= 0.5  # 疑似刷好评
    if user and user.positive_ratio < 0.1:
        weight *= 0.5  # 疑似恶意差评

    return raw_feedback * weight
```

### 7.4 转译结果生命周期

```
新提交 → trial（试用期）
  │
  ├── 30 天内未达 5 次使用 → deprecated（淘汰）
  │
  ├── 30 天内 avg_feedback < 0 → deprecated（淘汰）
  │
  └── 30 天内 usage ≥ 5 且 avg_feedback ≥ 0.5 → active（正式）
          │
          ├── 连续 90 天无使用 → inactive（休眠，但保留）
          │
          └── avg_feedback 连续 30 天 < 0 → deprecated（淘汰）
```

### 7.5 冷启动：新签名的处理

当签名首次出现（无精确/模糊匹配）：

1. **本地 Fallback**：使用 intent + domain 的默认模板（预置约 200 个）
2. **种子机制**：如果用户使用默认模板后给了正反馈，该用户可选择"分享我的转译"
3. **审核后入库**：分享的转译进入 trial 期，积累足够正反馈后变为 active

## 八、格式化器与 LLM 适配器

### 8.1 格式化器（结构化 → 文本）

把转译结果的 `structured_prompt` 转成目标 LLM 能理解的文本。

**Claude 最优格式**：

```
<task>
  <type>code_generation</type>
  <requirements>
    <item>使用 Python 语言</item>
    <item>包含异常处理</item>
    <item>包含反爬机制</item>
  </requirements>
  <constraints>
    <item>不使用第三方库</item>
    <item>添加详细注释</item>
  </constraints>
  <context>
    <user_expertise>intermediate</user_expertise>
  </context>
</task>

请根据以上要求生成代码。
```

**GPT 最优格式**：

```
你是一个编程助手。请按照以下要求生成代码：

【任务类型】代码生成
【要求】
1. 使用 Python 语言
2. 包含异常处理
3. 包含反爬机制
【约束】
1. 不使用第三方库
2. 添加详细注释
【用户水平】中级

请生成代码。
```

**DeepSeek 最优格式**：

```
# 任务：生成 Python 爬虫代码

## 要求
- 使用 Python
- 包含异常处理
- 包含反爬机制

## 约束
- 不使用第三方库
- 添加详细注释

## 用户水平：中级

## 输出
请直接输出完整代码。
```

### 8.2 适配器设计

```python
class LLMAdapter(ABC):
    @abstractmethod
    def format(self, structured_prompt: dict) -> str:
        pass

    @abstractmethod
    def get_system_prompt(self) -> str:
        pass

class ClaudeAdapter(LLMAdapter):
    def format(self, structured_prompt: dict) -> str:
        return xml_format(structured_prompt)

    def get_system_prompt(self) -> str:
        return "You are a helpful assistant. Follow the structured task instructions precisely."

class GPTAdapter(LLMAdapter):
    def format(self, structured_prompt: dict) -> str:
        return markdown_format(structured_prompt)

    def get_system_prompt(self) -> str:
        return "You are a helpful assistant."

class DeepSeekAdapter(LLMAdapter):
    def format(self, structured_prompt: dict) -> str:
        return markdown_format(structured_prompt, compact=True)

    def get_system_prompt(self) -> str:
        return ""
```

## 九、隐私与安全设计

### 9.1 不上传什么

| 不上传 | 原因 |
|--------|------|
| 原始用户输入 | 隐私核心，永不上传 |
| 对话全文 | 隐私核心，永不上传 |
| 个人身份信息 | 合规要求，永不上传 |
| 代码/文档内容 | 可能含敏感信息，永不上传 |

### 9.2 上传什么

| 上传 | 处理方式 |
|------|---------|
| 场景签名（六维度） | 结构化特征，无法反推原始输入 |
| 签名哈希 | 不可逆哈希 |
| 转译结果 ID | 不关联原始内容 |
| 隐式反馈值 | 匿名化用户哈希 |

### 9.3 匿名化机制

```python
import hashlib

def anonymize_user(user_id: str) -> str:
    """
    用户标识经设备密钥 + 盐值哈希后上传。
    即使云端也无法关联到真实用户。
    """
    device_salt = load_local_secret("device_salt")  # 首次启动生成，永不离开本地
    return hashlib.sha256(f"{user_id}:{device_salt}".encode()).hexdigest()[:16]
```

### 9.4 用户控制权

```yaml
privacy_settings:
  share_anonymized_data: true   # 是否参与集体网络（可随时关闭）
  share_personal_corrections: false  # 是否分享个人修正
  local_mode_only: false        # 完全离线模式（只用本地模板）
  data_retention_days: 90       # 云端保留匿名数据的天数
```

### 9.5 完全离线模式

当 `local_mode_only: true` 时：
- 跳过云端查询
- 仅使用本地缓存和预置模板
- 功能降级但可用

## 十一、风险分析

### 11.1 匹配稀疏性（高风险）

**问题**：六维度组合空间巨大，大多数签名可能没有历史记录。

**缓解**：
- 分层匹配（精确 → 模糊 → 粗粒度 → fallback）
- 预置 200+ 默认模板覆盖高频组合
- 向量相似度扩展匹配范围

### 11.2 冷启动（高风险）

**问题**：早期用户量少，云端数据库几乎为空。

**缓解**：
- Phase 1-2 先以本地模式运行，不依赖云端
- 预置模板覆盖 60% 场景
- 邀请种子用户快速积累数据

### 11.3 隐私合规（中风险）

**问题**：任何数据上传都可能引发隐私担忧。

**缓解**：
- 不上传原始输入（技术保证）
- 完全离线模式可选
- 透明的隐私面板，用户可见上传了什么

### 11.4 反馈刷量（中风险）

**问题**：恶意用户或自动化脚本刷好评/差评。

**缓解**：
- 用户可信度加权
- 异常检测（极端值比例）
- 新转译结果有试用期，不立即进入推荐

### 11.5 模型差异（中风险）

**问题**：对 Claude 最优的转译，对 DeepSeek 可能无效。

**缓解**：
- 按 model_family 分组存储和匹配
- 格式化器针对不同模型优化 prompt 格式
- 反馈也按 model_family 分离统计

### 11.6 过度拟合（低风险）

**问题**：系统过度推荐历史高赞转译，导致创新停滞。

**缓解**：
- 每次返回 top-3 而非 single best
- 新 trial 结果有一定曝光机会
- 定期评估"探索 vs 利用"比例
