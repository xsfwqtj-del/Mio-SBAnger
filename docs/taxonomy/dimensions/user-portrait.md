# 用户画像

> 维度类型：个人 | 实现方式：文本注入 | 验证方式：无验证

---

## 一、定义

**告诉 Agent 它在为谁服务——用户的身份、背景、当前状态。**

用户画像是 Agent 的"上下文锚点"。没有用户画像，Agent 会把用户当作"一个通用的匿名用户"——它对你的背景、偏好、当前状态一无所知。

### 控制什么

- **身份**：角色（开发者/产品经理/学生）、专业领域、经验水平
- **背景**：技术栈偏好、工作环境、语言
- **当前状态**：正在做什么项目、处于什么阶段

### 不控制什么

- Agent 怎么说话（那是交互风格）
- Agent 怎么思考（那是思维方式）
- 用户喜欢什么（那是偏好记忆——属记忆系统，不属行为配置）

---

## 二、注入位置

| 注入层 | 注入内容 | 注入方式 |
|--------|---------|---------|
| L2 人格层 | 用户身份 + 背景 | BUILD 阶段文本注入 system prompt |

用户画像属于 L2（volatile），因为不同项目的用户画像可能不同。全局 CLAUDE.md 写"我是一个开发者"，项目 CLAUDE.md 写"这个项目用 TypeScript + React"。

---

## 三、填写方法论

### 3.1 有效写法

**原则**：只写 Agent 需要知道的。不要写简历。

```
用户画像:
  角色: 独立开发者
  技术栈: Python, TypeScript, React
  当前项目: MioForever — 个人 AI 助手平台（使用中文交流）
  当前阶段: 早期设计验证
  偏好: TypeScript 优先于 Python（新项目），DeepSeek 作为主力模型
```

**为什么这些信息有用**：
- "独立开发者" → Agent 知道不需要考虑团队协作流程
- "早期设计验证" → Agent 知道方案可以粗，不需要完整的工程级设计
- "TypeScript 优先于 Python" → 选技术栈时不用再问

### 3.2 无效写法

```
# ❌ 差
用户画像: 资深全栈工程师，10 年经验，擅长 Java/Python/Go/Rust/C++，
曾经在 Google/Facebook 工作过，有三个孩子，喜欢攀岩。
```

问题：Agent 不需要知道你在哪里工作、几个孩子。这些信息占 token 但没有任何操作价值。

### 3.3 好的用户画像只包含可操作信息

**可操作** = Agent 在决策时可以直接使用这个信息。

| 信息 | 可操作？ | 说明 |
|------|---------|------|
| "3 年 Python 经验" | ✅ | 解释 Python 概念时可以不用从零开始 |
| "当前用 Windows" | ✅ | 命令用 PowerShell 而非 bash |
| "喜欢攀岩" | ❌ | 除非 Agent 在帮你做攀岩计划 |
| "Google 工作过" | ❌ | 对编程任务没有操作差异 |

---

## 四、验证方式

用户画像通常是纯建议性配置，无需验证。但如果用户画像写得太离谱（比如"我是一个专业级的"但写了很多垃圾代码），画像本身需要调整——这是用户自己的责任，不是 Agent 的验证。

## 五、场景模板

### 编程场景

```
用户画像（编程）:
  角色: {独立开发者 / 团队成员 / 技术负责人}
  专业技能: {语言/框架}，{经验水平}
  工作环境: {操作系统}，{编辑器/IDE}
  当前项目: {项目名 + 一句话描述}
  当前阶段: {原型验证 / 开发中 / 重构 / 维护}
  偏好: {技术栈偏好}，{风格偏好}
```

### 写作场景

```
用户画像（写作）:
  角色: {技术博客作者 / 产品写手 / 学术写作者}
  读者: {谁会读这些内容}
  风格偏好: {正式 / 轻松 / 技术性的}
  语言: {中文 / 英文 / 混合}
```

## 六、ECC 对标

ECC 中没有直接对应的"用户画像"配置。ECC 的配置是"怎么做事"，用户画像是"为谁做事"——前者是通用规则，后者是上下文。ECC 缺少用户画像可能是因为它设计为"通用配置包"，不假设用户身份。

**本维度补充的**：让 Agent 知道"我在为谁服务"。

---

## 七、常见陷阱

| 陷阱 | 为什么错 | 怎么改 |
|------|---------|--------|
| 写成了简历 | 大量不可操作的信息占 token | 每条画像问自己：Agent 在什么决策中会用到这个信息？ |
| 画像和交互风格混淆 | "我说话直接"不是画像，是交互风格 | 画像 = 我是谁，交互风格 = 我怎么和 Agent 说话 |
| 画像太具体 → 限制 Agent | "我只用 Vue" → Agent 在更好的方案面前不推荐 React | 偏好用"倾向"而非"只用" |
| 画像太久不更新 | "当前阶段: 设计"但实际已经在开发 | 定期检查用户画像是否过期 |
