# 出厂默认值

> 分类学第四轴：元属性 → 出厂默认 | 引用 MioForever 蓝图 〇.B.4 优先级

---

## 一、定义

**每项配置的 fallback 值——当用户从未设置此维度时，Agent 按什么规则运行。**

出厂默认确保 Agent 永远不会"裸奔"——即使没有任何用户配置，它也知道基本的行为边界。

---

## 二、九维度的出厂默认值

### 行为策略 — 出厂默认

```
安全红线:
  - 绝不执行 rm -rf 或类似递归删除命令
  - 绝不修改 .git 目录中的文件
  - 绝不执行 git push -f 到 main/master

操作纪律:
  - 修改文件前先读取当前内容
  - 不确定时停下来问（不猜）

边界:
  - 不做生产环境操作
```

### 检验回路 — 出厂默认

```
V1 参数校验: 始终开启（工具类型和必填字段检查）
回环终止: 工具回环 max=10，拦截回环 max=3
其他校验: 关闭（用户可按需开启）
```

### 推理模板 — 出厂默认

```
先结论后依据。第一句话就是答案。
```

### 工具可用性 — 出厂默认

```
权限: 所有工具默认 ask
纪律: 无特殊纪律
```

### 技能触发规则 — 出厂默认

```
触发: 用户手动触发优先，无自动语义匹配
冲突: 多个技能同时匹配 → 列出候选让用户选
```

### 用户画像 — 出厂默认

```
无。Agent 将用户视为"通用用户"，不做任何背景假设。
```

### 交互风格 — 出厂默认

```
语气: 直接。先结论后展开。
长度: 适中。复杂问题可以详细，简单问题简洁。
语言: 中文。
```

### 思维方式 — 出厂默认

```
原则:
  - 不确定就明确说明，不要猜
  - 多种方案同时列出，说明取舍
```

### 反思提示词 — 出厂默认

```
无。回答后不做额外自我检查。
```

---

## 三、出厂默认的设计原则

1. **安全优先**：不确定时默认拒绝（ask），不是默认放行
2. **最小约束**：只设安全底线和基本纪律，不给风格偏好（那是用户的事）
3. **可被覆盖**：任何出厂默认都可以被用户配置覆盖——出厂默认是 fallback，不是强制
4. **跨模型中立**：出厂默认不假设特定 LLM 的遵从特征
5. **面向所有用户**：出厂默认不是"我喜欢的默认值"，而是"大多数用户都能接受的起点"。不能把个人偏好伪装成通用出厂默认

---

## 四、与"烬汐居"的关系

MioForever 蓝图定义了"烬汐居"作为出厂配置的来源——用户在 CC 上运行烬汐居，总结规律，产出的配置作为 Mio 的出厂默认。

分类学中的出厂默认值 = 烬汐居的第一批产出物。后续可从 session-to-memory 的积累中迭代更新。
