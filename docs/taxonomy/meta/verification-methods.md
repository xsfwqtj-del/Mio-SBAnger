# 验证方法分类与选择指南

> 分类学第三轴：验证维度

---

## 一、六种验证方法

| 方法 | 速度 | 成本 | 确定性 | 适用 |
|------|------|------|--------|------|
| **规则匹配** | 毫秒 | 零 | 100% | 可正则化的规则 |
| **LLM 判定** | 秒级 | 调 flash | ~70% | 语义模糊的规则 |
| **格式校验** | 毫秒 | 零 | 100% | 结构化输出 |
| **行为拦截** | 毫秒 | 零 | 100% | 工具调用前 |
| **ECC 诊断** | 按需 | 零 | — | 配置失效排查 |
| **无验证** | — | — | — | 纯建议性配置 |

---

## 二、选择决策树

```
要检查的规则是？
  │
  ├── 能写成正则/关键词吗？ → 规则匹配
  │    例: "包含 rm -rf" → `r"rm\s+-rf"`
  │
  ├── 能在工具调用前检查吗？ → 行为拦截 (PRE_TOOL Hook)
  │    例: "不要 push force to main" → 检查 Bash 参数
  │
  ├── 输出有固定结构吗？ → 格式校验
  │    例: "方案必须包含风险分析段" → 检查输出是否含 `## 风险`
  │
  ├── 需要语义判断吗？ → LLM 判定 (调 flash)
  │    例: "回答是否多角度分析" → flash 判断
  │    注意: 成本高、确定性低。每 5-10 轮抽样一次
  │
  ├── 规则只是建议？ → 无验证
  │    例: "建议用中文回复"
  │
  └── 配置好像生效了但不确定？ → ECC 诊断
       检查 12 层栈中哪一层出了问题
```

## 三、按维度的验证方法推荐

| 维度 | 主要验证方法 | 辅助验证 |
|------|-------------|---------|
| 行为策略（安全红线） | 行为拦截 + 规则匹配 | — |
| 行为策略（操作纪律） | 行为拦截 | 规则匹配（关键词扫描） |
| 校验回路 | 行为拦截 | 审计日志 |
| 推理模板 | 格式校验 | LLM 判定（抽样） |
| 工具可用性 | 行为拦截（PRE_TOOL） | ECC 诊断 |
| 技能触发规则 | 规则匹配（触发日志） | ECC 诊断 |
| 用户画像 | 无验证 | — |
| 交互风格 | LLM 判定（抽样） | 规则匹配（长度检查） |
| 思维方式 | LLM 判定（抽样） | 反思提示词（自检） |
| 反思提示词 | LLM 判定（抽样） | — |

## 四、成本估算

假定每轮对话：pro 主力推理 + 可选 flash 验证。

| 验证配置 | 每轮增量成本 | 说明 |
|---------|------------|------|
| 仅规则匹配 + 行为拦截 | ~0 tokens | 本地毫秒级，零成本 |
| + 格式校验 | ~0 tokens | 同上 |
| + LLM 判定（每轮都做） | +500-1000 tokens/轮 | 成本翻倍，不推荐 |
| + LLM 判定（每 5 轮抽样） | +100-200 tokens/轮（均摊） | 推荐 |
| + LLM 判定（每 10 轮抽样） | +50-100 tokens/轮（均摊） | 最低成本 |

**原则**：能用本地规则解决的，不调 LLM。
