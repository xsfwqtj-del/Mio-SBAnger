# Agent 行为配置自检问卷

> 分类学 Phase 4 | 用这套问题检查你当前的 CLAUDE.md 覆盖了哪些、漏了哪些

---

## 使用方式

逐题回答。答"否"或"不确定"的项 → 优先补充。

---

## 通用维度

### 行为策略

1. □ Agent 知道绝对不能做什么吗？（安全红线）
2. □ 每条禁止操作有对应的替代方案吗？
3. □ Agent 在不确定时知道怎么做吗？（停下来问 / 默认拒绝）
4. □ 有灰色地带规则吗？（既不是白名单也不是黑名单的操作）
5. □ 行为策略和交互风格冲突时有明确的优先级吗？

### 校验回路

6. □ 工具调用前有参数检查吗？（V1）
7. □ Agent 有权对用户的方向性假设提出质疑吗？（V2）
8. □ 工具执行结果有验证吗？（V3/V4）
9. □ 回环有终止条件吗？（工具回环 max=10，拦截回环 max=3）
10. □ 违规操作有审计记录吗？

### 推理模板

11. □ Agent 的回答按什么结构组织？有明确的模板吗？
12. □ 不同任务类型（方案/施工/诊断）有不同的模板吗？
13. □ 模板数量 ≤5 条吗？（超过影响遵从率）

### 工具可用性

14. □ 每个可用工具有明确的"适用/不适用"场景吗？
15. □ 工具有三级权限分类吗？（allow/deny/ask）
16. □ 有工具使用纪律吗？（最多重试几次？同类工具有优先级吗？）
17. □ MCP 工具有启用/禁用条件吗？

### 技能触发规则

18. □ 每种技能有明确的触发条件吗？
19. □ 多个技能同时触发时知道谁优先吗？
20. □ 技能执行后知道回到什么状态吗？（主配置 / 覆盖）

---

## 个人维度

### 用户画像

21. □ Agent 知道你在什么角色下使用它吗？
22. □ Agent 知道你的技术栈偏好吗？
23. □ Agent 知道你当前项目的阶段吗？
24. □ 用户画像只有可操作信息吗？（不包含简历式无关信息）

### 交互风格

25. □ 你告诉 Agent 你喜欢的语气了吗？
26. □ 每条风格指令有反例吗？（"不要这样"的例子）
27. □ 风格和推理模板冲突时有明确的优先级吗？

### 思维方式

28. □ Agent 的决策原则明确吗？
29. □ 有具体的启发式指令吗？（"先审查再实现"而非"严谨"）
30. □ "不确定时"有明确的默认行为吗？

### 反思提示词

31. □ Agent 回答前有自我检查吗？
32. □ 反思问题是具体的可回答的吗？（不是"检查质量"）
33. □ 反思频率合理吗？（安全任务每轮，简单查询不做）

---

## 评分

- **≥28 个 ✅**：配置覆盖完整。只需维护。
- **20-27 个 ✅**：有明确的覆盖盲区。按优先级补充。
- **<20 个 ✅**：配置体系需要系统性建设。从行为策略和思维方式开始。
