# exp-01: 思维链回环 (self-check loop)

**日期**: 2026-06-11
**状态**: 完成
**问题**: 配置怎么让 LLM 视角更广、更中立、不盲从用户？

## 设计

配置 = 思维链注入 + 自检回环

1. 思维链（文本注入）：`先拆问题 → 查假设 → 多视角 → 不迎合 → 下结论` 写入 system prompt
2. 自检回环（验证）：LLM 回复后，拦截下来，拼接自检提示词再发给 LLM。规则是"先给结论了吗/指出假设了吗/多角度了吗/有不同意见主动提了吗"。LLM 回 PASS 放行，回修正版替换

## 为什么自检必须用 LLM 而不能用代码

检验项是语义层的——"是否指出了假设""是否从多角度分析"。正则和 JSON Schema 够不着。

## 发现

- 自检生效时，修正版明显更全面（多角度分析、指出假设）
- 但 LLM 不总遵守自检指令——有时先解释"不符合所有规则。修正如下：结论：…"再给修正版，而不是直接输出
- PASS 判定不稳定（同一条回复两次自检可能结果不同）
- 成本翻倍（每条消息调两次 LLM）
- 延迟翻倍（用户等两轮推理）
- 规则无场景感知（"自我介绍"不需要多角度分析，但规则照拦）

## 结论

语义层验证只能靠 LLM 本身，用代码（正则/JSON Schema）做不了。但 LLM 自检本身不可靠——需要找让自检更稳的办法，而不是退回正则。

下一步方向：让自检提示词更精确、减少 LLM 可自由发挥的空间、或改用更强的模型做自检（用 pro 审 flash）。
