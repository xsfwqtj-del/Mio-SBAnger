# 待办目录规范

## 文件命名

`[时间戳]-[分类]-[简述]-[已完成X-Y].md`

- **时间戳**：`YYYYMMDD-HHmmss`（创建时刻，精确到秒）
- **分类**：配置 / Mio / 技能 / 杂项
- **简述**：中文短语
- **进度**：`已完成N-M`，N=已完成数，M=总数。全部完成时 N=M

例：
- `20260610-053000-配置-精简ECC技能-已完成0-6.md`
- `20260610-053000-配置-精简ECC技能-已完成6-6.md`（全部完成）

## 文件格式

每份待办文件必须包含：

```markdown
# [标题]

## 现状（可选）
当前状态描述，供下次会话快速接上。

## 待执行
- [ ] 具体可执行的任务项

## 完成
- [x] 已完成的任务（附日期）
```

## 规则

1. **一个文件一个主题** — 不把互不相关的任务塞同一文件
2. **待办项要具体** — 每一行 [ ] 后面要能直接执行，不写"优化一下"这种
3. **完成项标注日期** — `- [x] (2026-06-10) 完成描述`
4. **现状段要精简** — 只写下次接上需要的上下文，不写长篇分析
5. **不改已完成项** — 完成后只往完成段追加，不修改历史
6. **进度同步** — 每完成一项，立即更新文件名末尾的 `已完成X-Y`，确保文件名始终反映当前进度
